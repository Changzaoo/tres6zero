# Direcao Visual

## Estilo

O visual deve ser:

- Moderno.
- Premium.
- Mobile-first.
- Sobrio.
- Inspirado em produtos Apple, mas sem copiar.
- Rico em microinteracoes leves.
- Claro para usuario comum.
- Sem excesso de informacao.
- Focado em operacao, nao em marketing exagerado.

## Paleta recomendada

- Fundo escuro quase preto.
- Branco para texto principal.
- Cinza frio para texto secundario.
- Azul e violeta como assinatura da marca.
- Verde para sucesso.
- Amarelo/ambar para alertas.
- Vermelho apenas para erro/perigo.

Evitar dominar a interface inteira com um unico tom de roxo ou azul. Usar contraste e respiro visual.

## Tipografia

- Titulos curtos, fortes e objetivos.
- Texto auxiliar pequeno, claro e util.
- Evitar paragrafos longos dentro de cards operacionais.
- Botoes com verbos diretos.

## Landing Page

- Hero forte com a marca SIX3° como sinal principal.
- Headline clara sobre videos 360.
- Demonstracao visual real do fluxo.
- Beneficios concretos: rapidez, QR Code, templates, galeria, leads.
- Prova social assim que houver usuarios reais.
- Planos com comparacao simples.
- CTA "Comecar a jornada".

## Dashboard

- Limpo e escaneavel.
- Cards com dados reais.
- Graficos so quando ajudam.
- Estados vazios orientando o primeiro passo.
- Sem excesso de cards decorativos.
- Priorizar a acao "Gravar" ou "Criar evento".

## Tela da funcao principal

- Deve parecer um editor compacto.
- Video/preview sempre visivel.
- Controles proximos do preview.
- Botoes menores e mais precisos.
- Timeline clara para efeito, template e trilha.
- Feedback visual em cada etapa.
- Resultado final com QR Code, "Ver video", "Compartilhar" e "Novo video".

## Templates

- Mobile precisa ter busca, filtros e preview.
- Templates devem mostrar retrato/paisagem/quadrado.
- Templates animados precisam responder a hover/touch.
- Evitar placeholders genericos no catalogo final.
- Organizar por tema: festa, casamento, aniversario, corporativo, viral, premium.

## Mobile

- Bottom navigation sempre acessivel.
- Sidebar lateral nao deve aparecer no mobile.
- Botao central "Gravar" deve ficar realmente centralizado.
- Conta/perfil deve estar acessivel no bottom bar.
- Topbar vazia nao deve aparecer.
- Cards precisam caber sem cortar badges ou textos.
- Evitar scroll desnecessario em telas operacionais.

## Componentes obrigatorios

| Componente | Direcao |
|---|---|
| Header | Compacto e sem redundancia |
| Sidebar | Desktop apenas |
| Bottom nav | Mobile, com acao principal clara |
| Cards | 8px a 16px quando card operacional; evitar cards dentro de cards |
| Botoes | Icone + texto quando comando claro |
| Inputs | Altura confortavel e foco visivel |
| Modais | Curtos, com acao principal evidente |
| Toasts | Feedback imediato |
| Skeleton loading | Usar para listas e dashboard |
| Empty states | Explicar proximo passo |
| Error states | Dizer o que fazer |
| Success states | Dar proxima acao |

## Microinteracoes

- Hover discreto em desktop.
- Touch states claros no mobile.
- Animações curtas, sem atrasar uso.
- Loading centralizado e suave.
- Previews de musica e templates devem ser leves.
- Evitar animações pesadas em listas grandes.

## Direcao por tela

### Login/cadastro

Visual limpo, logo grande, formulario curto, suporte acessivel e sem expor dados sensiveis.

### Planos

Cards com comparacao simples. No mobile, carrossel horizontal. Badge "Mais escolhido" sem corte. Plano atual deve aparecer no proprio card.

### Configuracoes

Organizar em secoes: perfil, tema, senha, dispositivos, notificacoes, seguranca e suporte. Deve lembrar tema claro/escuro/sistema.

### Admin

Mais denso, operacional, com filtros, clientes, suporte e notificacoes. Remover elementos decorativos desnecessarios.

## Regra de ouro visual

O usuario deve conseguir publicar o primeiro video sem ler documentacao externa. A interface deve conduzir pela propria ordem dos elementos.
